# 内测清单 · 上线门闩

> **目标**：先 **100 人技术内测** → **≤800 人小范围运营内测** → **公开上线运营**（最终百万会员）  
> **配套**：[运营手册](./million-member-ops-playbook-zh.md) · [稳定路线图](./stable-growth-roadmap-zh.md) · [pool-v4](./pool-v4-algorithm-zh.md)

---

## 1. 三扇门（必须按顺序过）

```mermaid
flowchart LR
  G0[门闩0 技术内测<br/>~20-50人 2-4周] --> G1[门闩1 小范围运营内测<br/>100-800人 1-3月]
  G1 --> G2[门闩2 公开上线<br/>阶段Ⅰ 日新增≤15]
  G2 --> G3[门闩3 规模扩张<br/>按路线图阶段Ⅱ+]
```

| 门闩 | 规模 | 周期 | 过了才能做什么 |
|------|------|------|----------------|
| **0 · 技术内测** | 20–50 人（含运营/技术） | 2–4 周 | 修 Bug、补未实现功能 |
| **1 · 运营内测** | 100 → **800** 人 | 1–3 月 | 验证付款率、转单链、护盘公式 |
| **2 · 公开上线** | 阶段Ⅰ 硬顶 **≤15/日** | 持续 | 对外拉新、服务中心正式接单 |
| **3 · 规模扩张** | 按 §2 阶段表提速 | — | 日新增硬顶上调（每次 +10% 需指标绿） |

**原则**：门闩没过 **禁止冲量**；宁可多测一周，不可带着 `pay_expired` 链路上线。

## 5. 门闩 2 · 公开上线运营

**过了门闩 1** 才能对外公开拉新；按 **阶段Ⅰ** 执行（日新增 **≤15**）。

### 5.1 上线前 Checklist（一项不过不上线）

**产品与算法**

- [ ] 转单链生产环境开启，非人工台账
- [ ] App 出场付款 + 转单通知（push/站内）
- [ ] pool-v4 配置从 GitHub/网关发布，用户可核对地址
- [ ] 运营看板可每日出数（App 或后台至少一种）

**组织与资金**

- [ ] 服务中心名单 + 链上地址备案
- [ ] 我方兜底 / 护盘地址池 + 备付金（L1/L2）
- [ ] 上级转单 SLA：24h 内响应
- [ ] 客服话术：转单规则、谁付谁收

**合规与风控**

- [ ] 日新增硬顶 **15** 写入运营规程
- [ ] 放缓 S1–S6 有人值班执行
- [ ] 无 P0 安全项（密钥、多签、地址篡改）

### 5.2 上线后 30 天观察指标（每日看 [运营手册](./million-member-ops-playbook-zh.md)）

| 指标 | 🟢 可维持 | 🟡 护盘 | 🔴 立刻放缓 |
|------|-----------|---------|-------------|
| 超时率（7日） | ≤0.8% | 0.8–1.2% | >1.2% |
| 积压率 | ≤18% | 18–25% | >25% |
| 临期未付 | 0 | 1–20 | >20 |
| 达标率（当月） | 90–110% | 80–90% | <80% 且付款恶化 |
| 护盘占比（7日） | 20–35% | 35–40% | >40% |

**上线后 30 天总评通过（方可宣布「正式上线稳定」并准备阶段Ⅱ提速评估）：**

| 条件 | 要求 |
|------|------|
| 30 天内 🔴 放缓触发 | **≤2 次** |
| 30 天平均超时率 | **≤0.8%** |
| 30 天平均积压率 | **≤20%** |
| 转单后付清率 | **≥99%** |
| 会员增长 | 符合阶段Ⅰ路径（约 **→3000/年** 节奏，允许滞后不可狂飙） |

## 8. 什么情况下 **禁止上线**

| 情况 | 原因 |
|------|------|
| 转单链未实现且无人工台账 | 会出现收不到款 |
| App 不能打出场地 | 付款率必崩 |
| 超时率内测 >1% | 上线必放量崩盘 |
| 回放多机不一致 | 用户看到的队与运营不一致 |
| 护盘可插队（任何特权） | 摧毁信心 |
| 无兜底备付金 | L3 转单无法闭环 |
| 日新增无硬顶管控 | 违反阶段Ⅰ |

## 10. 快速对照：内测什么 · 上线看什么

```
内测 = 功能对不对（T01-T23）+ 转单闭环 + 付款能点能验
上线 = 指标绿不绿（超时≤0.8%、积压≤18%、转单付清≥99%）+ 14/30 天连续达标
扩张 = 路线图阶段表 + 运营手册提速条件
```

**一句话**：**门闩 0 测准算法和转单，门闩 1 测准付款和护盘，门闩 2 看 30 天指标绿了才算真正上线运营。**
